# AI Repository Orientation

This repository documents a personal home lab environment.

The purpose is to allow an AI system to quickly understand the hardware, software,
and configuration of the system before assisting with troubleshooting or planning.

## Entry Points

Start with:

ai-context.md

This file contains a condensed snapshot of the system including network,
storage, services, and references to documentation.

## Repository Structure

notes/
Human-written documentation describing architecture, hardware, and system purpose.

outputs/hardware/
Automatically generated hardware inventory (CPU, devices, buses, radios, etc).

outputs/software/
Installed software inventory and service lists.

outputs/configs/
Safe configuration exports used for reproducibility and troubleshooting.

scripts/
Automation scripts that generate inventories and snapshots.

## Important Machines

Primary workstation:
Panasonic Toughbook CF-31 rugged laptop.

Primary server:
Dell Optiplex 7010 used as a home server and media server.

## Automation

System snapshots are generated by scripts in the scripts/ directory.
These scripts populate the outputs/ directories.

## Workflow Expectation

When assisting with this system, AI tools should:

1. Read ai-context.md first.
2. Use notes/ for architectural explanations.
3. Use outputs/ for factual system state.
4. Avoid assuming missing configuration—prefer referencing the exported data.
